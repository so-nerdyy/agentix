# Phase 10 - Canonical API Consolidation

## Purpose

Consolidate all runtime entrypoints onto a consistent contract.

## Scope

- Backend API.
- Shell bridge.
- Dashboard endpoints.
- Channel event payloads.

## Implementation

- Shared schemas.
- Versioned task and session payloads.
- Consistent error and approval responses.

## Acceptance Criteria

- UI, CLI, and channels speak the same task model.
- API changes are versioned and testable.
