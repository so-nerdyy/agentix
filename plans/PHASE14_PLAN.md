# Phase 14 - Hermes-Class Terminal UX

## Purpose

Deliver the command-line experience users expect from a Hermes-like product.

## Scope

- Banner and startup flow.
- Slash commands.
- Themes.
- Session history.
- Multiline input.
- Interrupt and retry ergonomics.

## Implementation

- Polished rendering.
- Clear command help.
- Consistent shell state and session persistence.

## Acceptance Criteria

- The terminal feels like a product, not a thin wrapper.
- Common shell actions are discoverable and fast.
