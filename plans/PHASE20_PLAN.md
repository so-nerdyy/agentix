# Phase 20 - Memory Productization

## Purpose

Make working memory a usable product surface instead of only an internal store.

## Scope

- Episodic, procedural, semantic, and working memory.
- Search and retrieval.
- Cross-surface visibility.

## Implementation

- Search APIs.
- Memory cleanup and consolidation.
- Workspace-scoped retention.

## Acceptance Criteria

- Users can inspect and search memory from the product surfaces.
- Memory stays consistent across shell, dashboard, and backend workflows.
