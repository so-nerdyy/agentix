# Phase 22 - Operator Task Detail and Approvals

## Purpose

Make task inspection and approval workflows complete enough for real operators.

## Scope

- Task detail view.
- Approval review.
- Step-by-step execution traces.

## Implementation

- Canonical task identity and drill-down.
- Approval prompts and decisions.
- Artifact and log inspection.

## Acceptance Criteria

- Operators can understand, approve, reject, and inspect work without leaving the product.
