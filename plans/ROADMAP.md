# Agentix Roadmap

This roadmap defines the intended end-state of Agentix as a Hermes-class agent platform with an Agentix-owned backend.

## Architecture Summary

- Hermes owns the user-facing shell, setup, model, update, cron, gateway, and command UX.
- Agentix owns orchestration, validation, approvals, memory, healing, the bridge/API, and Pi-worker execution.
- The product must work from a project folder with `agentix`, and it must also support server/dashboard and channel-driven workflows.

## Completion Bar

Agentix is complete when:

- the backend architecture is stable and documented,
- the shell and integrations are Hermes-class and installable,
- the dashboard, approvals, memory, and audit surfaces are functional,
- global install and local project workflows are reproducible,
- and the launch criteria in Phase 25 pass on a clean machine.
